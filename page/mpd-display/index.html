MPD Display

This page describes version 0.1

Introduction

MPD Display is a “now playing” display for the moOde audio player, other similar Raspberry-Pi-based music players, or any computer running the Music Player Daemon. It is intended to be used on a cheap/old/unused Android tablet and to be easily readable at variety of distances. Below is a screenshot, however it looks much cooler when the tablet is on top of your stereo cabinet!

The design philosophy is:

  • Show what’s playing, readable at a distance
  • Fit elegantly into a stereo rack
  • No album art
  • Avoid horizontal “marquee” scrolling

Top row controls are Progress, Previous Track, Play/Pause, Next Track, Appearance, Server Settings and “About” screen. Track title is first, then other information (which scrolls vertically if there isn’t enough space). There are 4 font themes each with a light and dark variant. The font size is set to be clearly legible from about 4 metres (12 feet) but can be tuned up and down for different distances and screen sizes. As well as the control buttons, tapping on the centre half of the screen toggles pause, tapping left or right skips to the previous or next track.

Pen icon means “Composer”, piano icon is “Performer” or “Artist” (depending on context) and vinyl icon is “Album”. Multi-value tags are supported. Tag your music library with the recommended Vorbis tags for best results. In particular for classical music use “artist” for the composer and “performer” for the orchestra/conductor/players. The “composer” tag is also supported.

The screenshot above is from the author’s 7" Kindle Fire but any Android tablet supported by Flutter should work (>= Android 4.4, although >= 4.1 might work). Larger tablets will work fine. Smaller devices like phones may work if you adjust the font size to suit and have good enough eyesight.

The app works best if you put the tablet in developer mode, connect it to the Raspberry Pi/computer, and use a supplied script to automatically unlock the screen and launch the app when playback begins. Here’s an overview diagram for moOde:

Required Hardware

  • A tablet (or phone).
  • A tablet stand. The author uses one of these but anything that can hold the tablet in a readable position is fine.
  • A Raspberry Pi or other computer running MPD.
  • A USB charge/sync cable compatible with your tablet

Required Software

The APK file and wakeup script installer discussed below, as well as the full source code, can be downloaded from the release page for version 0.1.

Setup Summary

  • Factory-reset your tablet and when you set it up don’t use any important accounts, especially not your main Google (or Amazon) account. Create a dummy account if required. This is because the wakeup script must be able to unlock the device without interaction. If it or the Raspberry Pi are stolen your passwords will be compromised.
  • Set the tablet display timeout to your preference. While playing MPD Display will keep the screen turned on so this mainly affects how long until the screen turns off when you pause or stop playback.
  • Enable developer mode on the tablet. Generic Android Instructions / Kindle Fire instructions. Don’t connect the tablet to USB yet.
  • If possible, assign a static IP to your Raspberry Pi on your router, then reboot it. This improves the chances that the tablet will always be able to find it on the network.
  • Enable SSH on the Raspberry Pi. This is enabled by default in moOde, with a default username and password. It must be manually enabled on Raspbian. For all other Linux systems it is almost certainly enabled by default.
  • Figure out the default username and password. For moOde this is pi and moodeaudio. For the rest of this document I’ll assume you’re using moOde. Adjust instructions if your setup is different.
  • SSH to the Raspberry Pi and install the Android Debug Bridge: sudo apt-get install adb. The version included with the operating system is fine, you don’t need the full Android development toolkit.
  • Enable adb debugging on the tablet.
  • Connect the tablet to the Raspberry Pi with the USB cable. A notification or dialog will appear asking you to authorize the Raspberry Pi. Be sure you authorize it before proceeding.
  • On the Raspberry Pi, run adb devices -l. It should list exactly one device. For my Kindle Fire it looks something like this:
pi@music:~ $ adb devices -l
List of devices attached
GCC0X010101010M8       device usb:1-1.2 product:mustang model:KFMUWI device:mustang transport_id:9
  • Copy the app APK file to the Raspberry Pi or to somewhere visible to the Raspberry Pi then install it to the tablet. If the APK was copied to the pi user then this is sufficient: adb install mpd-display-0.1.apk.
  • Manually open MPD Display on the tablet. By default it will try to connect to a server called “music” on the default port of 6600. Your Raspberry Pi will likely have a different name or IP address. Tap the connection button top-right and enter the correct name or IP address. You probably won’t need to change the port number from the default (6600). MPD Display should connect within a few seconds. If your player is stopped with an empty queue it will show only the server name. If it is paused or playing you’ll see the current track.
  • Make sure your player is paused or stopped then copy the wake-up script installer to the Raspberry Pi, or to somewhere visible to the Raspberry Pi and run it. If the script was copied to the pi user then this is sufficient: bash install-wakeup-mpd. This will install the script, a systemd service to control it, and enable and start it. By default the script assumes no password, and a swipe from bottom to top to unlock the display. You may have to edit the script to set the correct actions for your tablet. A fairly easy way to edit it is with nano /home/pi/wakeup-mpd-display.sh. Once you are finished editing run systemctl --user restart wakeup-mpd-display.
  • With the tablet connected by USB, start playback using your phone or computer. Within a couple of seconds the tablet screen should unlock and the app will start. adb can be a bit flaky sometimes so the script sometimes doesn’t work. Any suggestions on how to improve reliability are welcome.
